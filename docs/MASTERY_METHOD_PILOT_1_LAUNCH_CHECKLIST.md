# Pilot 1 Launch Checklist - Mastery Method

Status: ACTIVE — aligned to Mastery Method Naming Canon (Stage 2A)
Governing canon: [MASTERY_METHOD_NAMING_CANON.md](./MASTERY_METHOD_NAMING_CANON.md)
Owner: Kirsty / The Pressure Academy  
Applies to: Pilot 1 only

## Mission

Launch Pilot 1 in a controlled, low-chaos, high-trust way.

## Rule

Do not send the first family invite until all sections below are complete.

## Section A - Human Confirmation Gate

- [ ] Human Confirmation Checklist reviewed in full
- [ ] Kirsty approves Pilot 1 as ready to begin
- [ ] Pilot boundaries confirmed
- [ ] Controlled pilot only
- [ ] Minimal portal allocation only
- [ ] No broad rollout yet
- [ ] Parent-facing language approved
- [ ] Below-age placement language approved
- [ ] First-session success criteria agreed
- [ ] Session review discipline agreed

## Section B - Pilot Family Selection

- [ ] Pilot Student 1 chosen
- [ ] Family is low-complexity / low-risk
- [ ] Scorecard completed
- [ ] Parent concern is clear
- [ ] No major unresolved risk flags
- [ ] Suitable for a clean first pilot

## Section C - Placement + Curriculum Readiness

- [ ] Pilot Student Intake Template completed
- [ ] Student profile completed
- [ ] English placement chosen
- [ ] Mathematics placement chosen
- [ ] First 4 English lessons mapped
- [ ] First 4 Mathematics lessons mapped
- [ ] Kirsty approves the first 4+4 lesson chain
- [ ] Any manual-only resources identified
- [ ] Any portal-safe resources identified

## Section D - Portal Readiness

- [ ] Correct family token chosen
- [ ] Invite link verified in correct format: `/api/auth/activate?token=...`
- [ ] Approved pilot resources allocated
- [ ] Non-pilot resources withheld/manual
- [ ] Portal content checked before invite
- [ ] Resource order makes sense for Session 1

## Section E - Family Communication Readiness

- [ ] Welcome / orientation message drafted
- [ ] First parent update template prepared
- [ ] First CTA is clear and simple
- [ ] No over-explaining / over-giving
- [ ] Parent communication approved by Kirsty

## Section F - Session 1 Readiness

- [ ] Session 1 lesson selected
- [ ] Materials/resources ready
- [ ] Teacher prompts reviewed
- [ ] Expected student output understood
- [ ] Session Review Log ready to use immediately after lesson

## Section G - Go / No-Go

- [ ] All sections complete
- [ ] Kirsty gives final GO
- [ ] Family invite can now be sent

## After Session 1

- [ ] Session Review Log completed
- [ ] Parent Update sent
- [ ] Continue / adjust decision made for Session 2

## Operating Notes

- This checklist is an execution gate, not a suggestion list.
- Pilot 1 should remain intentionally narrow.
- If any item is incomplete, unresolved, or low-confidence, hold the invite and return the case to Kirsty review.
- Portal allocation for Pilot 1 should stay minimal and approved only.
- Use this checklist alongside the intake and deployment operating system and the portal access admin guide.
